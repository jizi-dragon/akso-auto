# 工作流创建与绑定 Spec

## Why
当前 `akso-basic-config-api` 对工作流模块的认知几乎为零（README 第八节标注"API 探索处于早期阶段"）。用户将实操演示一个完整的审批工作流创建过程，覆盖从新建工作流到绑定生命周期的全链路，通过 API 监听获取真实的端点、请求体和响应格式。

## What Changes
- 监听并记录工作流创建流程中所有 API 调用（端点、请求体、响应）
- 监听并记录工作流节点/步骤配置的 API
- 监听并记录工作流与生命周期绑定的 API
- 提取工作流数据结构的完整模式（基础信息、节点定义、连线、条件分支等）
- 输出工作流 API 行为分析文档，为后续工具化提供数据基础

## Impact
- Affected specs: 工作流（全新模块）
- Affected code: 暂无代码变更，本次仅做 API 学习与数据收集
- 产出文件: `output/workflow-monitor.log`、`output/workflow-api-analysis.md`

## ADDED Requirements
### Requirement: 工作流创建 API 监听
系统 SHALL 启动浏览器监听环境，捕获用户创建审批工作流全过程中的所有 API 调用，包括但不限于：
- 工作流基础信息保存
- 工作流节点/步骤的增删改
- 节点间连线/流转配置
- 审批节点配置（审批人、条件等）
- 工作流与生命周期状态的绑定

### Requirement: 交互式学习
监听过程中，遇到不明确的 API 行为或数据结构时，SHALL 立即向用户提问，确保理解准确。

### Requirement: 分析输出
完成监听后，SHALL 输出结构化分析文档，包含：
- API 端点清单与调用顺序
- 请求体/响应体数据结构
- 工作流节点类型与配置参数
- 生命周期绑定机制
