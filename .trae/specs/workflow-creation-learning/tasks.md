# Tasks

## 阶段 1：监听环境搭建

- [ ] Task 1: 搭建工作流 API 监听环境
  - 启动浏览器并登录 Akso
  - 启用全量网络请求/响应日志（聚焦 workflow/Workflow/Node/Condition/Branch 相关）
  - 等待用户进入工作流配置页面

## 阶段 2：工作流创建（交互式学习）

- [ ] Task 2: 监听"创建工作流基础信息"的 API
  - 用户创建新工作流，填写名称、code 等基本信息
  - 捕获创建工作流的 API 端点和请求体

- [ ] Task 3: 监听"添加审批节点"的 API
  - 用户在工作流画布中添加审批节点
  - 捕获节点创建的 API 端点、节点数据结构

- [ ] Task 4: 监听"配置审批人"的 API
  - 用户在节点中配置审批人（指定角色/用户）
  - 捕获审批人配置的数据结构

- [ ] Task 5: 监听"节点连线/流转配置"的 API
  - 用户连接节点、配置流转条件
  - 捕获连线/边/条件的 API 和数据结构

- [ ] Task 6: 监听"工作流发布/启用"的 API
  - 用户发布或启用工作流
  - 捕获发布 API

## 阶段 3：生命周期绑定

- [ ] Task 7: 监听"工作流绑定生命周期"的 API
  - 用户将工作流绑定到生命周期的状态切换上
  - 捕获绑定 API 和相关参数

## 阶段 4：分析输出

- [ ] Task 8: 分析所有捕获数据，输出分析文档
  - 整理 API 端点清单（按调用顺序）
  - 提取数据结构模式（节点、连线、审批配置等）
  - 输出到 `output/workflow-api-analysis.md`
  - 更新 README.md 第八节

# Task Dependencies
- Task 2~7 均依赖 Task 1（需先搭建监听环境）
- Task 3~5 依赖 Task 2（工作流创建后才能加节点）
- Task 6 依赖 Task 3~5（节点配置完成后发布）
- Task 7 依赖 Task 6（发布后才能绑定）
- Task 8 依赖 Task 2~7（需所有数据）
