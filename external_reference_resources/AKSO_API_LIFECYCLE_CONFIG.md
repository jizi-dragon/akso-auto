编制日期：2026-07-08

## 一、配置方案总览
### 1.1 设计目标
在AKSO eGMP系统中建立原料药（API）全生命周期阶段闸门（Stage-Gate）管理体系，实现：
• Gate 0~5 六个阶段闸门的线上化管理
• 技术转移全流程的数字化控制
• QA Gate Keeper 独立质量监督权的系统保障
• 交付物审批、准出条件检查、决策日志的完整电子记录
• 与现有偏差、CAPA、变更控制模块的无缝衔接

### 2.2 生命周期设计
生命周期名称：原料药生命周期项目生命周期

状态定义：
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│  Gate 0  │───▶│  Gate 1  │───▶│  Gate 2  │───▶│  Gate 3  │───▶│  Gate 4  │───▶│  Gate 5  │───▶│  已关闭   │
│ 立项启动  │    │ 开发规划  │    │ 准备度确认│    │ 工艺演示  │    │ 验证就绪  │    │ 商业投产  │    │          │
└──────────┘    └──────────┘    └──────────┘    └──────────┘    └──────────┘    └──────────┘    └──────────┘
     │                │                │                │                │                │
     └────────────────┴────────────────┴────────────────┴────────────────┴────────────────┘
                                          │
                                     ┌────┴────┐
                                     │  已暂停  │
                                     └─────────┘

状态流转规则：
- Gate 0 → Gate 1：条件 = Gate 0评审通过
- Gate 1 → Gate 2：条件 = Gate 1评审通过
- Gate 2 → Gate 3：条件 = Gate 2评审通过
- Gate 3 → Gate 4：条件 = Gate 3评审通过
- Gate 4 → Gate 5：条件 = Gate 4评审通过
- Gate 5 → 已关闭：条件 = Gate 5评审通过 + 项目关闭报告批准
- 任意阶段 → 已暂停：QA或项目经理可触发
- 已暂停 → 原阶段：恢复时回到暂停前阶段

状态权限：
- Gate N → Gate N+1 的流转：仅QA负责人可执行（需电子签名）
- 暂停/恢复：QA负责人或项目经理
- 已关闭：仅QA负责人

### 3.2 各Gate准出条件配置
**Gate 0 准出条件：**
1. 商业论证已通过管理层审批
2. 初始风险评估已完成且风险可接受
3. 法规影响已评估并形成记录
4. 质量资源已确认到位
5. 变更控制已获QA批准
6. 原料药起始物料定义已初步明确
7. （预留扩展）
**Gate 1 准出条件：**
8. 技术转移方案已获QA批准
9. 控制策略已明确并经QA批准
10. CQA/CPP已识别并形成文件
11. 验证总计划（VMP）已批准
12. 分析方法策略已确认
13. 杂质谱评估已完成（含ICH M7评估）
14. （预留扩展）
**Gate 2 准出条件：**
15. 关键设备/设施确认合格
16. 清洁验证方案已批准
17. 关键SOP已生效且人员已完成培训
18. 批生产记录已批准
19. 原材料已验证合格
20. Site Readiness Review结论为通过
21. 实验室分析方法已验证/确认
**Gate 3 准出条件：**
22. 工程批/转移批已按计划完成
23. 所有偏差已调查并关闭或已制定CAPA
24. 分析方法转移报告已批准
25. 中期风险评估结论为风险可控
26. 杂质谱与研发预期一致
27. （预留扩展）
28. （预留扩展）
**Gate 4 准出条件：**
29. PPQ已完成且报告批准
30. 验证总结报告已获QA批准
31. 清洁验证已获批准
32. 可比性研究已完成且结论可接受
33. 初始稳定性数据符合预期
34. 商业化标准已建立（质量标准、SOP、批记录）
35. （预留扩展）
**Gate 5 准出条件：**
36. 技术转移最终报告已获QA批准
37. 项目已正式关闭，残留行动计划已完成
38. CPV计划已建立并投入运行
39. 经验教训已形成文件并纳入知识库
40. APQR/PQR机制已就绪
41. 所有变更控制已完成关闭
42. （预留扩展）
### 3.3 生命周期
生命周期名称：阶段闸门评审生命周期

状态：待组织 → 进行中 → 已完成 → 已关闭
                     ↓
                  需补充材料 → 进行中（重新评审）

审批流程：
- 评审完成需QA电子签名确认
- "通过"结论触发项目状态推进到下一Gate
- "不通过"结论触发行动项创建
